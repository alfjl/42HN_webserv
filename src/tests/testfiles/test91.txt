		<h1 class="entry-title">HTTP Headers: Understanding HTTP Part 2</h1>
	</header><!-- .entry-header -->

	<div class="entry-content clearfix">

		<p><img class="size-thumbnail wp-image-14277 alignleft" src="http://www.steves-internet-guide.com/wp-content/uploads/HTTP-Headers-1-100x100.jpg" alt="HTTP-Headers" width="100" height="100" />In the second part of our short http course we will look at http headers.</p>
<p>If you&#8217;ve missed part 1 then here it is-<a href="http://www.steves-internet-guide.com/http-basics/" target="_blank" rel="noopener noreferrer"> Part 1- http basics</a><span id="more-5722"></span></p>
<h3>HTTP Headers</h3>
<p>HTTP headers re used to convey additional information between the client and the server.</p>
<p>Although they are optional they make up the most of the http request and are almost always present.</p>
<p>When you request a web page using a web browser the headers are <strong>inserted automatically</strong> by the web browser, and you don&#8217;t see them.</p>
<p>Similarly the response headers are inserted by the web server and are not seen by the user.</p>
<p>There are extensions available for both Firefox and Chrome that let you view http headers and also command line tools like curl.</p>
<p>We will look at some example request and response headers later.</p>
<h3>Request and Response Header Structure</h3>
<p>Request and response headers share a common structure.</p>
<p>They consist of a<strong> header name + colon + header value</strong>. Example</p>
<p><img loading="lazy" class="size-full wp-image-5714 alignnone" src="http://www.steves-internet-guide.com/wp-content/uploads/HTTP-Headers.jpg" alt="HTTP-Headers" width="392" height="222" srcset="http://www.steves-internet-guide.com/wp-content/uploads/HTTP-Headers.jpg 392w, http://www.steves-internet-guide.com/wp-content/uploads/HTTP-Headers-300x170.jpg 300w" sizes="(max-width: 392px) 100vw, 392px" /></p>
<p>An header can have multiple values, in which case they are separated using a comma.</p>
<p><strong>Field names</strong> are <strong>case insensitive</strong> according to the <a href="https://www.w3.org/Protocols/rfc2616/rfc2616-sec4.html" target="_blank" rel="noopener noreferrer">RFC 2616</a> but <strong>field values</strong> should be treated as <strong>case sensitive</strong></p>
<p>There are many headers, and it is not important to be familiar with them all.</p>
<p>There is a really good list of headers with explanation on <a href="http://www.tutorialspoint.com/http/http_header_fields.htm" target="_blank" rel="noopener noreferrer">tutorialspoint</a></p>
<p>You should note that only necessary headers are sent all other headers are assumed by the web server and client to be their default.</p>
<p>For example the connection header is not normally sent as the default behaviour is <strong>keep-alive</strong> and this is assumed by the server.</p>
<h3>Common Request Headers</h3>
<h4>Connection Header</h4>
<p>The original HTTP protocol used<strong> non persistent</strong> connections.</p>
<p>This meant that the client</p>
<ol>
<li>Made a request</li>
<li>Got a Response</li>
<li>Closed the Connection</li>
</ol>
<p>If you consider a TCP/IP connection to be the same as a telephone connection. This means that you:</p>
<ol>
<li>Dial the number and get an answer (connection established.</li>
<li>Say something and get an acknowledgement</li>
<li>Hang Up.</li>
</ol>
<p>Because it takes time and resources to <strong>establish the connection</strong> in the first place it makes no sense to drop it so quickly.</p>
<p>Therefore in<strong> HTTP 1.0</strong> the client can tell the server that it will keep the connection open by using the <strong>connection: keep-alive</strong> <strong>header</strong>.</p>
<blockquote><p>In <strong>HTTP v1.1</strong> the default behaviour was changed and <strong>persistent connections</strong> become the <strong>default mode.</strong></p></blockquote>
<p>Now the client can tell the server that it will <strong>close the connection</strong> by using the header <strong>connection: close.</strong></p>
<p><em><div class="outliney">This header is not normally sent as the default assumed by the server is <strong>keep-alive.</div></strong></em></p>
<h4>Host Header</h4>
<p>Almost all websites,including this one, use shared hosting.</p>
<p>With shared hosting the web server is configured as a virtual host and all virtual hosts will be assigned to a <strong>single IP address</strong>.</p>
<p>The<strong> hosts header</strong> tells the web server which server to refer the request to e.g.</p>
<p><strong>Host:www.steves-internet-guide.com</strong></p>
<h4>User-Agent Header</h4>
<p>This gives information about the client making the request as shown below:</p>
<p><img loading="lazy" class="alignnone wp-image-14248" src="http://www.steves-internet-guide.com/wp-content/uploads/user-agent-header.jpg" alt="user-agent-header" width="764" height="24" srcset="http://www.steves-internet-guide.com/wp-content/uploads/user-agent-header.jpg 725w, http://www.steves-internet-guide.com/wp-content/uploads/user-agent-header-300x10.jpg 300w" sizes="(max-width: 764px) 100vw, 764px" /></p>
<h3>Accept Request Headers</h3>
<p>These headers are used for <strong>content negotiation</strong> and are sent by the client (browser) to the server, and tells the server what formats the client can understand.</p>
<p>The <strong>Accept header</strong> is used to tell the server what <strong>media types</strong> the client prefers e.g. Text, audio etc.</p>
<p>For normal web pages common values are <strong>text/plain</strong> and <strong>text/html.</strong></p>
<p>e.g.</p>
<p><strong>Accept:text/plain,text/html</strong></p>
<p>For <strong>JSON</strong> encoded data the header</p>
<p><strong>Accept:application/json</strong></p>
<p>is used.</p>
<p>Other accept headers are</p>
<ul>
<li><strong>Accept-Charset:ISO-8859,UTF-8</strong></li>
<li>Accept-Language:en-GB,en</li>
<li><strong>Accept-Encoding</strong>:<strong>gzip,deflate</strong></li>
</ul>
<p>Example Headers from live session:</p>
<p><img loading="lazy" class="alignnone size-full wp-image-14253" src="http://www.steves-internet-guide.com/wp-content/uploads/Request-Headers-Example.jpg" alt="Request-Headers-Example" width="610" height="153" srcset="http://www.steves-internet-guide.com/wp-content/uploads/Request-Headers-Example.jpg 610w, http://www.steves-internet-guide.com/wp-content/uploads/Request-Headers-Example-300x75.jpg 300w" sizes="(max-width: 610px) 100vw, 610px" /></p>
<h3>Common Response Headers</h3>
<p>The screen shot below shows the response headers using the curl command requesting a web page from <strong>steves-internet-guide.com</strong>:</p>
<p><img loading="lazy" class="alignnone size-full wp-image-14254" src="http://www.steves-internet-guide.com/wp-content/uploads/response-headers.jpg" alt="response-headers" width="700" height="316" srcset="http://www.steves-internet-guide.com/wp-content/uploads/response-headers.jpg 700w, http://www.steves-internet-guide.com/wp-content/uploads/response-headers-300x135.jpg 300w" sizes="(max-width: 700px) 100vw, 700px" /></p>
<p>The first line of the response is mandatory and consists of the protocol (<strong> HTTP/1.1</strong>),response code (<strong>200</strong>)and description (<strong>OK</strong>).</p>
<p>All subsequent lines are optional</p>
<p>The headers shown are:</p>
<p><strong>CONTENT-Type</strong> -This is <strong>Text/html</strong> which is a web page. It also includes the character set which is UTF-8.</p>
<p><strong>Connection</strong> &#8211; Is keep-alive which means that the connection is held open.</p>
<p><strong>Keep-Alive</strong> -This setting as shown is a timeout which says that the server will keep the connection open for <strong>15 seconds</strong> then close it. See <a href="https://blog.stackpath.com/glossary-keep-alive/" target="_blank" rel="noopener noreferrer">here</a> for more details.</p>
<p>All of the other headers are self explanatory.</p>
<p>Here is another screen shot (partial) of a site returning <strong>JSON data</strong> , notice the Content type.</p>
<p><img loading="lazy" class="alignnone size-full wp-image-14259" src="http://www.steves-internet-guide.com/wp-content/uploads/response-header-json.jpg" alt="response-header-json" width="752" height="249" srcset="http://www.steves-internet-guide.com/wp-content/uploads/response-header-json.jpg 752w, http://www.steves-internet-guide.com/wp-content/uploads/response-header-json-300x99.jpg 300w" sizes="(max-width: 752px) 100vw, 752px" /></p>
<h3>Setting Headers</h3>
<p>For normal users headers are of no interest as they are hidden and created automatically either by the browser (request headers) or web server (response  headers).</p>
<p>However web developers and IOT developers will need to be able to set these headers manually.</p>
<p>The most common tool used for this is the command line <a href="http://www.steves-internet-guide.com/using-curl-iot-applications/" target="_blank" rel="noopener noreferrer">curl utility</a> but extensions are available for Google Chrome and Firefox that also let you set<strong> request headers.</strong></p>
<p>Just do a search for <strong>chrome headers extension</strong> or <strong>Firefox headers add-on.</strong></p>
<h3>Summary</h3>
<p>HTTP headers convey addition information between the client and web server and are used on the request and response.</p>
<p>Although they are inserted automatically by the web browser and web server it is important for web developers,IOT developers and engineers to have a basic understanding of them as it is sometimes necessary to manually add headers when making <strong>API requests</strong> using command line tools like curl.</p>
<p>It is also sometimes necessary to process response headers in IOT applications.</p>
