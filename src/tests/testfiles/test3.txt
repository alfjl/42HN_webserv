<h1 id="what-are-http-status-codes">What are HTTP status codes?</h1>
</div>
</div>
</div></div></section><section class="section" data-row-theme="no-theme" data-padding="is-paddingless-top"><div class="umb-container">
<div class="columns">
<div class="column is-6 grid-section__cell" >
<div class="content grid-section__cell-content">
<p>HTTP status codes are three-digit responses from the server to the browser-side request. Everyone has probably gotten the classic 404 page-not-found error. That is an HTTP client error status code and there are a lot more of them.</p>
<p>These status codes (also called response status codes) serve as a means of communication between the server and the internet browser and there are multiple code classes based on the type of information they are communicating. The differences in classes are indicated through the first digit of the error code, for example: just like a 404, any other 4xx will mean that in some way the page or website could not be reached, while a 2xx means that your request was successfully completed.</p>

<h2 id="http-status-code-categories">How are HTTP status codes categorized?</h2>
<p>HTTP status codes are split into 5 different categories. Each category will give you hints as to what the response was, even if you don't know the specific response code.</p>
<p>For an explanation of each category - and each individual status code - click on the corresponding link below or go to our <a href="#http-status-codes-list">complete list of HTTP status codes</a>.</p>
<ul>
<li><a href="#1xx">1xx - Informational: The server has received the request and is continuing the process</a> </li>
<li><a href="#2xx">2xx - Successful: The request was successful and the browser has received the expected information</a> </li>
<li><a href="#3xx">3xx (Redirection): You have been redirected and the completion of the request requires further action</a></li>
<li><a href="#4xx">4xx (Client Error): The website or the page could not be reached, either the page is unavailable or the request contains bad syntax</a> </li>
<li><a href="#5xx">5xx (Server Error): While the request appears to be valid, the server could not complete the request</a></li>
</ul>

<h2 id="http-status-codes-explained">HTTP Status Codes explained individually</h2>
<p>In some cases a HTTP response code might be descriptive enough to understand its meaning. 200 OK probably means that everything went okay. But what about a 103 Early Hints, 205 Reset Content and 305 Use Proxy?</p>
<p><strong>Below is an explanation for all 63 status codes, sorted in the 5 overall categories</strong>.</p>

<h2 id="1xx">What does a 1xx Informational status code mean?</h2>
<p>A 1xx Informational status code means that the server has received the request and is continuing the process. A 1xx status code is purely temporary and is given while the request processing continues. For most tasks you won't encounter these much, as it's not the final response to the request.</p>
<ul>
<li><a href="#100-continue">100 Continue</a></li>
<li><a href="#101-switching-protocols">101 Switching protocols</a></li>
<li><a href="#102-processing">102 Processing</a></li>
<li><a href="#103-early-hints">103 Early Hints</a></li>
</ul>
<p> </p>
<h3 id="100-continue"><strong>What does 100 Continue mean?</strong></h3>
<p>The <strong>100 Continue</strong> status code means that the initial part of the request has been received by the server and that the client should proceed with the request or ignore the response if the request has already finished.</p>
<p> </p>
<h3 id="101-switching-protocols"><strong>What does 101 Switching protocols mean?</strong></h3>
<p>The <strong>101 Switching protocols</strong> status code means that the server understands the Upgrade header field request and indicates which protocol it is switching to.</p>
<p> </p>
<h3 id="102-processing"><strong>What does 102 Processing mean?</strong></h3>
<p>The <strong>102 Processing</strong> status code means that the server has accepted the full request but has not yet completed it and no response is available as of yet.</p>
<p> </p>
<h3 id="103-early-hints"><strong>What does 103 Early Hints mean?</strong></h3>
<p>The <strong>103 Early hints</strong> status code is intended to be used to allow the user agent to preload resources, while the server prepares a response. It is intended to be primarily used with the Link Header.</p>

<h2 id="2xx">What does a 2xx Succesful status code mean?</h2>
<p>A 2xx Succesful status code means that the request was successful and the browser has received the expected information. This is generally the one you want to see, as it means that the request was a success and has been received, understood and accepted it. As a website owner you should make sure that all pages and resources (images, videos, etc.) all return a 2xx status code. This means that browsers can reach it successfully and that your website visitors can see and use your website.</p>
<ul>
<li><a href="#200-ok">200 OK</a></li>
<li><a href="#201-created">201 Created</a></li>
<li><a href="#202-accepted">202 Accepted</a></li>
<li><a href="#203-non-authoritative-information">203 Non-Authoritative Information</a></li>
<li><a href="#204-no-content">204 No Content</a></li>
<li><a href="#205-reset-content">205 Reset Content</a></li>
<li><a href="#206-partial-content">206 Partial Content</a></li>
<li><a href="#207-multi-status">207 Multi-Status</a></li>
<li><a href="#208-already-reported">208 Already Reported</a></li>
<li><a href="#226-im-used">226 IM Used</a></li>
</ul>
<p> </p>
<h3 id="200-ok"><strong>What does 200 OK mean?</strong></h3>
<p>The <strong>200 OK</strong> status code means that the request was successful, but the meaning of success depends on the request method used:</p>
<ul>
<li>GET: The requested resource has been fetched and transmitted to the message body.</li>
<li>HEAD: The header fields from the requested resource are sent in without the message body. </li>
<li>POST or PUT: A description of the result of the action is transmitted to the message body.</li>
<li>TRACE: The request messages, as received by the server, will be included in the message body</li>
</ul>
<p>When looking at things SEO-wise the 200 OK response code is the perfect status code for a functioning page, all the linked pages are working as they should. A 200 will mean that search engine crawlers can successfully crawl the page and it will be put into their search index.</p>
<p> </p>
<h3 id="201-created"><strong>What does 201 Created mean?</strong></h3>
<p>The <strong>201 Created</strong> status code means that the request was successfully fulfilled and resulted in one or possibly multiple new resources being created.</p>
<p> </p>
<h3 id="202-accepted"><strong>What does 202 Accepted mean?</strong></h3>
<p>The <strong>202 Accepted</strong> status code means that the request has been accepted for processing, but the processing has not been finished yet. The request may or may not be completed when the processing eventually takes place.</p>
<p> </p>
<h3 id="203-non-authoritative-information"><strong>What does 203 Non-Authoritative Information mean?</strong></h3>
<p>The <strong>203 Non-Authoritative Information</strong> status code means that the request was successful. However, the meta-information that has been received is different from the one on the origin server and has instead been collected from a 3rd party or local copy. When not used for backups or mirrors of another resource a 200 OK response is preferable.</p>
<p> </p>
<h3 id="204-no-content"><strong>What does 204 No Content mean?</strong></h3>
<p>The <strong>204 No Content</strong> status code means that while the server has successfully fulfilled the request, there is no available content for this request. But the user agent might want to update its currently cached headers for this resource, for the new one. </p>
<p> </p>
<h3 id="205-reset-content"><strong>What does 205 Reset Content mean?</strong></h3>
<p>The <strong>205 Reset Content</strong> status code means that the user should reset the document that sent this request.</p>
<p> </p>
<h3 id="206-partial-content"><strong>What does 206 Partial Content mean?</strong></h3>
<p>The <strong>206 Partial Content</strong> response code is a response to a Range header sent from the client when requesting only a part of the resource.</p>
<p> </p>
<h3 id="207-multi-status"><strong>What does 207 Multi-Status mean?</strong></h3>
<p>The <strong>207 Multi-Status</strong> status code conveys information about multiple resources, in situation when multiple status codes are appropriate.</p>
<p> </p>
<h3 id="208-already-reported"><strong>What does 208 Already Reported mean?</strong></h3>
<p>The <strong>208 Already Reported</strong> status code is used inside the response element <em>DAV: propstat</em>, in order to avoid enumerating the internal members of multiple bindings to the same collection repeatedly.</p>
<p> </p>
<h3 id="226-im-used"><strong>What does 226 IM Used mean?</strong></h3>
<p>The <strong>226 IM response</strong> code means that the server has successfully fulfilled a <em>GET request</em> for the resource, and the response is a representation of the result of one or multiple instance-manipulations applied to the current instance.</p>

<h2><strong>What does a 3xx Redirection code mean?</strong></h2>
<p>A 3xx Redirection status code means that you have been redirected and the completion of the request requires further action. Redirects are a natural part of the internet and you shouldn't be scared to have 3xx redirect status codes on your website. A redirect means that the request was received successfully, but that the resource was found elsewhere. If a webpage has changed path and you try to access it through the old path, your CMS will often redirect the user to the new path. Ultimately the request will end in a 2xx success, but first it must go through the 3xx redirection.</p>
<ul>
<li><a href="#300-multiple-choices">300 Multiple Choices</a></li>
<li><a href="#301-moved-permanently">301 Moved Permanently</a></li>
<li><a href="#302-found">302 Found (Previously "Moved temporarily")</a></li>
<li><a href="#303-see-other">303 See Other</a></li>
<li><a href="#304-not-modified">304 Not Modified</a></li>
<li><a href="#305-use-proxy">305 Use Proxy</a></li>
<li><a href="#306-switch-proxy">306 Switch Proxy</a></li>
<li><a href="#307-temporary-redirect">307 Temporary Redirect</a></li>
<li><a href="#308-permanent-redirect">308 Permanent Redirect</a></li>
</ul>
<p> </p>
<h3 id="300-multiple-choices"><strong>What does 300 Multiple Choices mean?</strong></h3>
<p>The <strong>300 Multiple Choices</strong> status code means that the request has multiple possible responses and the user/user agent should choose one.</p>
<p> </p>
<h3 id="301-moved-permanently"><strong>What does 301 Moved Permanently mean?</strong></h3>
<p>The <strong>301 Moved Permanently</strong> response code means that the target resource has been assigned a new permanent URL and any references to this resources in the future should use one of the URLs included in the response.</p>
<p>When looking at things SEO-wise the 301 Permanent Redirect should be used every time a URL is moved permanently. This redirect passes your current link equity from your content to the new URL. Links that result in a status code 301 does give slightly less link equity than 200. So if you have a lot of links going through a 301 Permanent Redirect it is advised to fix these, if possible.</p>
<p> </p>
<h3 id="302-found"><strong>What does 302 Found (Previously “Moved temporarily”) mean?</strong></h3>
<p>The <strong>302 Found</strong> status code, previously known as “Moved temporarily”, means that the URI of the request has been changed temporarily, and since changes can be made to the URI in the future, the effective request URI should be used for future requests.</p>
<p>When looking at things SEO-wise the 302 Found should only be used when making temporary changes as it does not pass the link equity the same way as a 301. If the page is not going to come back you should always use 301.</p>
<p> </p>
<h3 id="303-see-other"><strong>What does 303 See Other mean?</strong></h3>
<p><strong>The 303 See Other</strong> response code is sent by the server in order to direct the client to get the requested resource at another URI with a GET request.</p>
<p> </p>
<h3 id="304-not-modified"><strong>What does 304 Not Modified mean?</strong></h3>
<p>The <strong>304 Not Modified</strong> response code informs the client that the response has not been modified. This means that the client can continue to use the already present, cached version of the response.</p>
<p> </p>
<h3 id="305-use-proxy"><strong>What does 305 Use Proxy mean?</strong></h3>
<p>The <strong>305 Use Proxy</strong> status code instructs a client that it should connect to a proxy and then repeat the same request there. This response code is deprecated due to security concerns.</p>
<p> </p>
<h3 id="306-switch-proxy"><strong>What does 306 Switch Proxy mean?</strong></h3>
<p>The <strong>306 Switch proxy</strong> status code is no longer in use. It was used to inform the client that the subsequent requests should use the specified proxy.</p>
<p> </p>
<h3 id="307-temporary-redirect"><strong>What does 307 Temporary Redirect mean?</strong></h3>
<p>The <strong>307 Temporary Redirect</strong> status code gets sent by the server in order to direct the client to the requested resource at another URI. The request method, however, must not be changed.</p>
<p> </p>
<h3 id="308-permanent-redirect"><strong>What does 308 Permanent Redirect mean?</strong></h3>
<p>The <strong>308 Permanent Redirect</strong> status code means that the requested resource has been permanently assigned a new URI and future references to the resource should be made by using one of the enclosed URIs.</p>

As a website owner you should do your best to avoid these, as it means your users will not find what they're looking for. This can be either pages that are no longer found and are either temporarily or permanently gone. Besides giving a bad user experience, it can also hurt your <a href="#http-status-codes-seo">SEO efforts</a>.</p>

<h3 id="400-bad-request"><strong>What does 400 Bad Request mean?</strong></h3>
<p>The <strong>400 Bad Request</strong> status code means that the server could not understand the request because of invalid syntax.</p>
<p> </p>
<h3 id="401-unauthorized"><strong>What does 401 Unauthorized mean?</strong></h3>
<p>The <strong>401 Unauthorized</strong> status code means that the request has not been applied because the server requires user authentication.</p>
<p> </p>
<h3 id="402-payment-required"><strong>What does 402 Payment Required mean?</strong></h3>
<p>The <strong>402 Payment Required</strong> status code is a response reserved for future use. It was originally created to be implemented in digital payment systems, however, it is rarely used and a standard convention of using it does not exist.</p>
<p> </p>
<h3 id="403-forbidden"><strong>What does 403 Forbidden mean?</strong></h3>
<p>The <strong>403 Forbidden</strong> status code means that the client request has been rejected because the client does not have rights to access the content. Unlike a 401 error, the client's identity is known to the server, but since they are not authorized to view the content, giving the proper response is rejected by the server.</p>
<p> </p>
<h3 id="404-not-found"><strong>What does error 404 mean?</strong></h3>
<p>The <strong>404 Not Found</strong> status code means that the server either did not find a current representation for the requested resource or is trying to hide its existence from an unauthorized client.</p>
<p>When looking at things SEO-wise the 404 Not Found status code pages with a high volume of traffic should be redirected using a 301 to the most relevant page possible. For some pages, however, a 404 might be necessary, for example, if the product is out of stock for an extended period of time. If you have external links pointing to a page that returns 404, you will lose the link equity those links would otherwise give.</p>
<p><a href="#fix-404-errors">If you need to fix 404 errors, jump to this section</a>.</p>
<p> </p>
<h3 id="405-method-not-allowed"><strong>What does 405 Method Not Allowed mean?</strong></h3>
<p>The <strong>405 Method Not Allowed</strong> status code means that while the server knows the request method, the method has been disabled and can not be used.</p>
<p> </p>
<h3 id="406-not-acceptable"><strong>What does 406 Not Acceptable mean?</strong></h3>
<p>The <strong>406 Not Acceptable</strong> status code is sent by the server when it does not find any content following the criteria given by the user agent.</p>
<p> </p>
<h3 id="407-proxy-authentication-required"><strong>What does 407 Proxy Authentication Required mean?</strong></h3>
<p>The <strong>407 Proxy Authentication Required</strong> status code means that the client must first be authenticated by a proxy (similar to a 401).</p>
<p> </p>
<h3 id="408-request-timeout"><strong>What does 408 Request Timeout mean?</strong></h3>
<p>The <strong>408 Request Timeout</strong> status code means that the server did not receive a complete request in the time that it prepared to wait.</p>
<p> </p>
<h3 id="409-conflict"><strong>What does 409 Conflict mean?</strong></h3>
<p>The <strong>409 Conflict</strong> status code means that the request could not be fulfilled due to a conflict with the current state of the target resource and is used in situations where the user might be able to resubmit the request after resolving the conflict.</p>
<p> </p>
<h3 id="410-gone"><strong>What does 410 Gone mean?</strong></h3>
<p>The <strong>410 Gone</strong> status code means that the target resource has been deleted and the condition seems to be permanent. </p>
<p>When looking at things SEO-wise the 410 Gone status code is a more permanent version a 404. The page will no longer be available from the server and has no forwarding address available. If you want to completely remove a page from Googles search index, then using 410 on a page is the proper way of doing it (instead of simply 404). </p>
<p> </p>
<h3 id="411-length-required"><strong>What does 411 Length Required mean?</strong></h3>
<p>The <strong>411 Length Required</strong> status code means that the server has rejected the request because it requires the Content-Length header field to be defined.</p>
<p> </p>
<h3 id="412-precondition-failed"><strong>What does 412 Precondition Failed mean?</strong></h3>
<p>The <strong>412 Precondition Failed</strong> status code means the server does not meet one or multiple preconditions that were indicated in the request header fields.</p>
<p> </p>
<h3 id="413-payload-too-large"><strong>What does 413 Payload Too Large mean?</strong></h3>
<p>The <strong>413 Payload Too Large</strong> status code means the server refuses to process the request because the request payload is larger than the server is able or willing to process. While the server may close the connection to prevent the client from continuing the request, it should generate a Retry-After header field and after how long can the client retry.</p>
<p> </p>
<h3 id="414-uri-too-long"><strong>What does 414 URI Too Long mean?</strong></h3>
<p>The <strong>414 URI Too Long</strong> status code means that the server is refusing to service the request because the request-target was longer than the server was willing to interpret.</p>
<p> </p>
<h3 id="415-unsupported-media-type"><strong>What does 415 Unsupported Media Type mean?</strong></h3>
<p>The <strong>415 Unsupported Media Type</strong> status code means that the server is rejecting the request because it does not support the media format of the requested data.</p>
<p> </p>
<h3 id="416-range-not-satisfiable"><strong>What does 416 Range Not Satisfiable mean?</strong></h3>
<p>The <strong>416 Range Not Satisfiable</strong> status code means that the range specified in the Range header field of the request can't be fulfilled. The reason might be that the given range is outside the size of the target URI's data.</p>
<p> </p>
<h3 id="417-expectation-failed"><strong>What does 417 Expectation Failed mean?</strong></h3>
<p>The <strong>417 Expectation Failed</strong> status code means that the Expectation indicated by the Expect request-header field could not be met by the server.</p>
<p> </p>
<h3 id="418-im-a-teapot"><strong>What does 418 I’m a Teapot mean?</strong></h3>
<p>The <strong>418 I'm a Teapot</strong> status code means that the server refuses to brew coffee because it is, in fact, a teapot. (It is a reference to a 1998 April Fools' joke called ''Hyper Text Coffee Pot Control Protocol'').</p>
<p> </p>
<h3 id="421-misdirected-request"><strong>What does 421 Misdirected Request mean?</strong></h3>
<p>The <strong>421 Misdirected Request</strong> status code means that the client request was directed at a server that is not configured to produce a response.</p>
<p> </p>
<h3 id="422-unprocessable-entity"><strong>What does 422 Unprocessable Entity mean?</strong></h3>
<p>The <strong>422 Unprocessable Entity</strong> status code means that while the request was well-formed, the server was unable to follow it, due to semantic errors.</p>
<p> </p>
<h3 id="423-locked"><strong>What does 423 Locked mean?</strong></h3>
<p>The <strong>423 Locked</strong> status code means that the resource that is being accessed is locked.</p>
<p> </p>
<h3 id="424-failed-dependency"><strong>What does 424 Failed Dependency mean?</strong></h3>
<p>The <strong>424 Failed Dependency</strong> status code means that the request failed due to the failure of a previous request.</p>
<p> </p>
<h3 id="425-too-early"><strong>What does 425 Too Early mean?</strong></h3>
<p>The <strong>425 Too Early</strong> status code means that the server is not willing to risk processing a request that might be replayed.</p>
<p> </p>
<h3 id="426-upgrade-required"><strong>What does 426 Upgrade Required mean?</strong></h3>
<p>The <strong>426 Upgrade Required</strong> status code means that while the server refuses to perform the given request using the current protocol, it might be willing to do so after the client has been upgraded to a different protocol.</p>
<p> </p>
<h3 id="428-precondition-required"><strong>What does 428 Precondition Required mean?</strong></h3>
<p>The <strong>428 Precondition Required</strong> status code means that the origin server requires the request to be conditional.</p>
<p> </p>
<h3 id="429-too-many-requests"><strong>What does 429 Too Many Requests mean?</strong></h3>
<p>The <strong>429 Too Many Requests</strong> response code means that in the given time, the user has sent too many requests.</p>
<p> </p>
<h3 id="431-request-header-fields-too-large"><strong>What does 431 Request Header Fields Too Large mean?</strong></h3>
<p>The <strong>431 Request Header Fields Too Large</strong> means that the server is not willing to process the request because its header fields are indeed too large, however, the request may be submitted again once the size of the request header fields is reduced.</p>
<p> </p>
<h3 id="451-unavailable-for-legal-reasons"><strong>What does 451 Unavailable For Legal Reasons mean?</strong></h3>
<p>The <strong>451 Unavailable For Legal Reasons</strong> response code means that the user has requested an illegal resource (such as pages and sites blocked by the government).</p>

<h2 id="http-status-codes-seo">HTTP Status Codes &amp; SEO: Here are the ones you need to know</h2>
<p>If you want great results with your SEO it's important to work with <a href="/umbraco-for-marketers/umbraco-the-seo-friendly-cms/#technical-seo-features" data-anchor="#technical-seo-features">technical SEO</a>. A big part of that is handling response codes on your website to ensure that the website is properly crawled by Googlebot and that your content returns the proper response code when it is requested.</p>
<p>Below are the 5 status codes that you need to know as an SEO.</p>

<h3>200 OK &amp; SEO</h3>
<p>This is the goal for 99 % of your content (pages, media, etc.): a successful status code that means everything works like it should. This is critical to a well-functioning website and for a great user experience.</p>
<p>It also gives you the reassurance that all external pointing to your website will give you link value. If all of your content returns a 200 OK status code you can rest assured that your website is working and is properly accessible for crawlers and visitors alike.</p>
<p> </p>
<h3>301 Moved Permanently &amp; SEO</h3>
<p>In a perfect world all of your content will stay on the same URL and always respond with a 200 OK. But that's rarely how it works in the real world when managing a website.</p>
<p>If a page has changed its URL you will need to set up a redirect to send users and bots from the old URL to the new URL. Otherwise they will be met by a 404 page (see explanation below). In some <a href="/knowledge-base/cms/">content management systems</a>, automatic redirect is a <a href="/umbraco-for-marketers/umbraco-the-seo-friendly-cms/">built-in SEO feature</a>, while in others you'll have to manually set it up.</p>
<p>You should use 301 redirects if the page is permanently gone and will not come back. This is the case for pages that have changed their URL or content that has been deleted.</p>
<p>If something is gone, instead of simply moved, you can either let it become a 404 or choose to redirect it to something similar on your website. This could be a product that you're no longer going to sell, where it can make sense to redirect that page to the category page instead.</p>
<p> </p>
<h3>302 Found &amp; SEO</h3>
<p>What if the page is only gone temporarily, but will come back again later? To give a good user experience you don't want it to become a 404 page and you also don't want to make a permanent 301 redirect.</p>
<p>The solution is to use 302 Found instead of a 301. To the user it will be the same, but to search engine crawlers and bots, it will tell them to check back later on, as this redirect is only temporary. That means the old URL will keep its value while it's gone.</p>
<p>This is perfect for pages that will only be unavailable temporarily. An example could be a sold-out product which is removed from the website until it is back in stock. In that case you want to make use of a 302 Found temporary redirect. Once the product is back online, you remove the redirect and the page will have kept its value (and good rankings in search engines).</p>
<p>Note that if the 302 redirect is there for too long, Google will consider it a 301 permanent redirect instead.</p>
<p> </p>
<h3>404 Not Found &amp; SEO</h3>
<p>If a page is no longer found it will result in a 404 page (<em>psst, <a href="/blog/how-do-you-make-a-great-404-page/">here's some tips on how to make a great 404 page</a></em>). This means, that the server tells users, crawlers and bots that the page they were looking for is not found. </p>
<p>These are important to keep an eye on, as they can hurt your SEO a lot if not monitored and fixed.</p>
<p>Firstly, if a page returns a 404 it won't be shown in Google's search results. While it won't get removed instantly, it will be after a short period of time, if not fixed. So if any of the pages that give you organic traffic end up returning a 404 error, you need to fix it quickly. See some <a href="#fix-404-errors" data-anchor="#fix-404-errors">tips on how to fix 404 errors here</a>.</p>
<p>Secondly, if the page had any external links pointing to it, they will no longer give any value to your website. So even if the page does not receive any organic traffic, it can hurt your SEO performance.</p>
<p>The easiest way to fix this problem is to set up a 301 redirect. This will give a better user experience and will pass most of the link value from external links to the new page you're redirecting to.</p>
<p> </p>
<h3>410 Gone &amp; SEO</h3>
<p>What if you actually want a page to completely disappear from Google's search engine index? It might be intuitive to simply let it return a 404 error, but there's one caveat with that:</p>
<p><em>404 does not tell crawlers and bots why the page was not found</em>. For that reason, Google still might keep thinking that the page is there, even if it encounters a 404 error. In most cases this is fine, but there's one case where you want to make it crystal clear that the content is gone. And that's if you have been hacked and malicious content has been added to your website.</p>
<p>After doing cleanup it's not enough to let the URLs return 404. Instead you should make sure that they return the 410 Gone status code. This clearly tells crawlers and bots that the page is gone for good and that they should remove it from their index.</p>
<p> </p>
<h3>5xx Server errors &amp; SEO</h3>
<p>The last one is not directly SEO-related, but more of a good tip for troubleshooting your website.</p>
<p>If you see any response code that start with 5xx, you should instantly know that it is a server error. This will help you greatly when it comes to troubleshooting and fixing it. </p>
<p>Instead of wasting time trying to fix the problem through your CMS backend or elsewhere, you can go straight to fixing the server. No matter if you do your own hosting or if you use a hosting provider, it is helpful to know that it's related to the server, not your website.</p>

</div></div></section><section class="section" data-row-theme="is-grey" data-padding="is-paddingless-bottom"><div class="umb-container">
<div class="columns">
<div class="column is-6 grid-section__cell" >
<div class="content grid-section__cell-content">
<h2 id="fix-404-errors">How to fix 404 errors</h2>
<p>If you have seen the status code 404 Page Not found you might have been thinking <em>"What does error 404 mean?"</em></p>
<p>HTTP status code 404 means "Page Not Found". This means that the request you sent was received by the server, but it could not find the page you were looking for.</p>
<p>So to fix the 404 error you first need to find the cause for it, which can be due to many things. But since you know that the server is reachable (otherwise it would have returned a 5xx error), you know that the error is client-side. Basically that means it's your fault (not as harsh as it sounds).</p>
<p>Luckily it means that you can also fix the error on your end without having to debugging your server or reaching out to your hosting provider.</p>
</div>
</div>
<div class="column is-6 grid-section__cell" data-vertical-position="Middle" data-horizontal-position="Center">
<div class="content grid-section__cell-content">

</div>
</div>
</div></div></section><section class="section" data-row-theme="is-grey" data-padding="is-paddingless-top"><div class="umb-container">
<div class="columns">
<div class="column is-12 grid-section__cell" >
<div class="content grid-section__cell-content">
<p><strong>Run through this checklist to fix the 404 error:</strong></p>
<ol>
<li>Refresh the page. It might seem simple, but a 404 error could have just been a temporary issue that is fixed simply by trying again.<br /><br /></li>
<li>Check the URL you typed in. Did you make a mistake somewhere? If the URL you typed is not exactly right a 404 is to be expected (unless a 3xx redirect is set up)<br /><br /></li>
<li>If no mistake was made, try again from a different device like your mobile phone or tablet. If it works from a different device it's most likely due to browser cache and cookies. Clear the browser cache and delete cookies, then try to access the page again.</li>
</ol>
<p> </p>
<p>If none of the above tricks worked, it's likely that the piece of content was deleted or some other way moved. If the content was moved, which caused it to change URL, then you should make sure to set up a redirect that points from the old URL to the new one. That way you're sure that anyone looking for the page can still find it.</p>
<p>If it was deleted, then a 404 error is the right response as the page truly was not found on your website and everything is working as it should.</p>
<p> </p>
<p><strong>If you see a lot of your visitors ending up on this page, then it's most likely due to one of 3 things:</strong></p>
<ol>
<li>You have an internal link on your website pointing to the page. If that's the case, you should remove (or change) this link so it no longer points to a page that is not found.<br /><br /></li>
<li>External websites link to the page. This is trickier as you can't simply change it (although you can reach out to the websites and ask them to change it). A more simple thing for you to do would be to add a 301 redirect and point from the page to a different page on your website. That way you lead people to a page that is working, while also being a <a href="#http-status-codes-seo">SEO-friendly option</a>.<br /><br /></li>
<li>The page is still showing up in Google's search results and sending traffic to your website. While this is usually short-lived, it is possible that a 404 page can stay in the search results for a while, as a 404 does not tell Googlebot whether the page is temporarily or permanently missing. If this is happening, you'll need to get the page back quickly or redirect it to a different page, so you won't lose your rankings in Google.</li>
</ol>
</div>
</div>
</div></div></section><section class="section" data-row-theme="is-grey" data-padding="is-paddingless-top"><div class="umb-container">
<div class="columns">
<div class="column is-12 grid-section__cell" >
<div class="content grid-section__cell-content">
<h2>Find 404 pages in Google Analytics</h2>
<p>Have you ever wondered if there’s a way to find 404 pages in Google Analytics?</p>
<p>Well, the good news is, there are ways to find them within your analytics setup. Now this can sometimes come down to how your website is configured. One way to find them is to check by page title - Your 404 page will very likely have “404” or “Page not found” in the title. It’s very easy to find the 404 page this way:<br /><br /></p>
<ul>
<li>Within your analytics tool, go to behavior → Site Content → All Pages.</li>
<li>Now you can set your primary dimension to Page title and search for  “404” or “Page not Found”.</li>
<li>And now - you should hopefully have your 404 report.</li>
</ul>
<p><br />Now this is not the only way to set it up, but there is a very good chance that you can find the pages this way. <br /><br /></p>
</div>
</div>

<h2 id="fix-503-errors">How to fix 503 errors</h2>
<p>The 503 response code means "Service Unavailable" and happens when a server is currently not ready to handle the request. This can be either due to the server being down for maintenance or if it is overloaded.</p>
<p>If this error occurs on your own website, you need to have a look at your server, as something is making it unable to process the requests made.</p>
<p>Unfortunately it's not always easy and simple to fix. A 503 response code can be due to many things, but at least you know it's server-related and not client-side like with a 404 error.</p>
</div>
</div>
</div></div></section><section class="section" data-padding="is-paddingless-top"><div class="umb-container">
<div class="columns">
<div class="column is-12 grid-section__cell" >
<div class="content grid-section__cell-content">
<p><strong>Here's a list of steps you can go through to identify and fix a 503 error:</strong></p>
<ol>
<li>Is your website still running and receiving traffic? Check your Analytics tools or server logs to see if other visitors are getting through to your website.<br /><br /></li>
<li>If it's no longer receiving traffic your server might be under maintenance or have crashed. If it's under maintenance then a 503 is to be expected and everything will work fine once it's done. If it crashed you should try restarting it.<br /><br /></li>
<li>If you're receiving huge amounts of website traffic, the server is most likely overloaded and returning 503 errors because it doesn't have the resources to keep up. Before fixing such an issue you need to identify it the traffic is from real visitors or if you're the victim of a <a href="https://en.wikipedia.org/wiki/Denial-of-service_attack">DoS or DDoS attack</a>.<br /><br /></li>
<li>Identifying a DoS or DDoS attack can be hard, but the most effective ways are:
<ol>
<li>Checking if one or more IP addresses make a lot of requests</li>
<li>The TTL (time to live) on a ping request times out</li>
<li>Analyzing the server logs and seeing huge spikes in traffic<br /><br /></li>
</ol>
</li>
<li>If it looks to be a DoS or DDoS attack, you'll need to apply one or more <a href="https://en.wikipedia.org/wiki/Denial-of-service_attack#Defense_techniques" data-anchor="#Defense_techniques">defense techniques</a> to stop the attack. <br /><br /></li>
<li>If it is not a DoS or DDoS attack, then your website has most likely become more popular than your server can handle. The best way to fix such an issue is to look at connection limits, bandwidth throttling, system resources, fail-safes that might have triggered or anything that might be limiting server performance. Essentially you'll need to upgrade your server so it has the proper resources to handle the traffic.</li>
</ol>
<p> </p>
<p>The above list is great for troubleshooting one-off 503 errors. But if it happens on a regular basis, then it might be a more permanent problem with your server that you should dive deeper into fixing. This can be inefficient processes using up all of your resources or your server simply not having enough allocated resources to handle the traffic your website is receiving.</p>
</div>
</div>
</div></div></section><section class="section" ><div class="umb-container">
<div class="columns">
<div class="column is-6 grid-section__cell" >
<div class="content grid-section__cell-content">
<h2>What status code is returned by a website when the browser request is successful?</h2>
<p>With all these HTTP status codes - It can be a bit daunting to figure if any of them are good. But some of them are. <br />A browser will return a 2xx status code if the browser request was successful. So a 2xx code is the one you want to see. The 200 status code means that the browser’s request was successful and received, understood, and accepted.</p>
</div>
